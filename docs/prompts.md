# Claude Code 단계별 프롬프트
## 페이레터 캐릭터 테스트 플랫폼 구현 가이드

---

## STEP 1 — 프로젝트 뼈대 + 화면 전환 시스템

```
다음 조건에 맞는 웹 프로젝트 뼈대를 생성해 줘.

[프로젝트 개요]
- 페이레터(Payletter) 브랜드의 MBTI 스타일 캐릭터 진단 테스트 웹사이트
- 순수 HTML5 + CSS3 + Vanilla JavaScript (ES6+)만 사용 (프레임워크 없음)
- 백엔드 없이 프론트엔드 단독으로 동작하는 정적 사이트

[파일 구조]
index.html
admin.html
css/style.css
js/data.js
js/engine.js
js/app.js
js/admin.js

[index.html 요구사항]
- 3개의 화면 섹션을 하나의 HTML에 포함: #screen-intro / #screen-question / #screen-result
- 초기에는 #screen-intro만 display:block, 나머지는 display:none
- 한국어 최적화 폰트 스택 적용: 'Apple SD Gothic Neo', 'Malgun Gothic', 'Noto Sans KR', sans-serif
- 모바일 뷰포트 메타태그 필수

[css/style.css 요구사항]
- 모바일 우선(Mobile First) 반응형 설계
- 최대 콘텐츠 너비 520px, 수평 중앙 정렬
- 색상 변수 CSS Custom Properties로 정의:
  --color-primary: #5B4FE8
  --color-secondary: #FF6B9D
  --color-bg: #F0F4FF
  --color-card: #FFFFFF
  --color-text: #1A1A2E
  --color-muted: #666680
- 카드형 UI: border-radius 24px, box-shadow 부드럽게
- 버튼 hover/active 트랜지션 효과 포함

[js/app.js 요구사항]
- showScreen(screenId) 함수: 모든 화면 숨기고 해당 화면만 표시 + fadeIn 애니메이션
- DOMContentLoaded 시 localStorage에 'pl_result' 키 확인
  - 있으면: 인트로에 "이전 결과 보기" 버튼 추가 노출
  - 없으면: CTA 버튼("테스트 시작하기")만 노출
- 전역 appState 객체 초기화:
  { currentScreen: 'intro', currentQuestion: 0, answers: [], result: null }

결과물: 위 파일 구조 전체를 생성하고, 화면 전환이 정상 동작하는지 확인할 수 있도록 인트로 화면에 임시 "테스트 시작하기" 버튼을 만들어 #screen-question으로 이동하는 것까지 구현해 줘.
```

## STEP 2 — 질문 데이터 + 점수 계산 엔진

```
js/data.js와 js/engine.js를 완성해 줘.

[js/data.js — 질문 데이터]
아래 10개 질문을 JavaScript 배열(QUESTIONS)로 정의해 줘.
각 질문 객체 구조:
{ id, question, options: { A: { text, weights: {...} }, B: { text, weights: {...} } } }

가중치 키 이름: swift / careful / people / task / flexible / structured / idea / execution

질문 목록:
Q1. 새로운 아이디어가 떠올랐을 때 나는
  A. 바로 공유해본다 → swift+1, people+1
  B. 정리한 뒤 공유한다 → careful+1, task+1

Q2. 회의에서 의견을 낼 때 나는
  A. 떠오르는 생각을 자유롭게 말한다 → swift+1, idea+1
  B. 생각을 정리해서 말한다 → careful+1, execution+1

Q3. 협업할 때 나는
  A. 다양한 사람과 소통하며 진행한다 → people+1
  B. 맡은 역할을 중심으로 진행한다 → task+1

Q4. 프로젝트가 시작되면
  A. 여러 방법을 시도하며 방향을 찾는다 → flexible+1, idea+1
  B. 계획과 구조를 먼저 세운다 → structured+1, execution+1

Q5. 정보를 전달할 때 나는
  A. 빠르게 공유하는 편이다 → swift+1
  B. 정확하게 정리해서 전달한다 → careful+1

Q6. 업무 진행 방식은
  A. 상황에 맞게 유연하게 진행한다 → flexible+1
  B. 계획에 맞게 진행한다 → structured+1

Q7. 문제를 해결할 때 나는
  A. 다양한 가능성을 먼저 생각한다 → idea+1
  B. 현실적인 해결을 바로 실행한다 → execution+1

Q8. 새로운 사람과 협업할 때 나는
  A. 먼저 대화를 시작한다 → people+1, swift+1
  B. 필요한 내용 중심으로 이야기한다 → task+1, careful+1

Q9. 아이디어를 발전시킬 때 나는
  A. 여러 아이디어를 연결한다 → idea+1
  B. 실행 가능한 방법을 구체화한다 → execution+1

Q10. 업무 스타일은
  A. 다양한 방법을 시도한다 → flexible+1
  B. 정해진 방식대로 진행한다 → structured+1

[js/engine.js — 점수 계산 + 결과 매핑]
calculateResult(answers) 함수를 구현해 줘.
- answers: ['A','B','A',...] 길이 10짜리 배열
- 각 답변에 해당하는 weights를 누적 합산
- 4축 승자 결정 (타이 브레이킹 규칙 엄수):
  swift > careful → 'S' / careful >= swift → 'C'
  people > task → 'P' / task >= people → 'T'
  flexible > structured → 'F' / structured >= flexible → 'S'
  idea > execution → 'I' / execution >= idea → 'E'
- 4자리 코드 생성 후 CHARACTERS 맵에서 캐릭터 객체 반환
- 유효하지 않은 코드 방어 로직 포함 (console.error + null 반환)
- getAxisSummary(answers) 함수도 추가:
  각 축의 점수와 퍼센트를 반환 (결과 화면 축 요약 칩에 사용)

결과물: data.js와 engine.js 완성 후, 브라우저 콘솔에서
calculateResult(['A','A','A','A','A','A','A','A','A','A']) 실행 시
올바른 캐릭터 객체가 반환되는지 확인할 수 있도록 임시 테스트 코드도 포함해 줘.
```
